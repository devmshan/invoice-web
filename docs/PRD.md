# invoice-web MVP PRD

## 핵심 정보

**목적**: 프리랜서/업체가 노션에 입력한 견적서를 고객이 고유 URL로 접속해 웹에서 확인하고 PDF로 다운받을 수 있게 한다.
**사용자**: 견적서를 보내는 프리랜서/1인 업체 (발행자), 견적서를 받아 확인하는 클라이언트 (수신자)

---

## 사용자 여정

```
[발행자 흐름]

1. 노션 데이터베이스
   ↓ 발행자가 견적서 항목 입력 (발행자 정보, 수신자 정보, 항목, 금액 등)

2. 노션 페이지 생성 완료
   ↓ 노션 페이지 ID 확보

3. 견적서 URL 생성
   → 고유 URL 패턴: /quote/[notion-page-id]
   ↓ URL을 고객에게 전달 (이메일, 메신저 등 직접 전달)

---

[수신자(고객) 흐름]

1. 견적서 URL 접속
   ↓ 노션 API로 데이터 실시간 조회

   데이터 존재 → 견적서 뷰 페이지 표시
   데이터 없음 → 오류 페이지 표시
   ↓

2. 견적서 웹 뷰 페이지
   ↓ 견적서 내용 확인 (발행자 정보, 수신자 정보, 항목 목록, 금액 계산, 유효기간)

3. PDF 다운로드 버튼 클릭
   ↓ 브라우저 인쇄 기능 또는 puppeteer 기반 PDF 생성

4. PDF 파일 저장 완료
```

### 4. 비기능 요구사항 (NFR)

| ID          | 항목          | 요구사항                                                                                                                                   |
| ----------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **NFR-001** | API 캐싱      | React `cache`로 요청 내 중복제거 + `unstable_cache`로 60초 TTL 크로스 요청 캐싱 적용. 동일 견적서 60초 내 재요청 시 Notion API 호출 0회    |
| **NFR-002** | Rate Limiting | `/quote/*` 경로에 IP 기반 슬라이딩 윈도우 방식 (1분당 30회 제한). 초과 시 429 응답 + `Retry-After: 60` 헤더                                |
| **NFR-003** | 구조화된 로깅 | pino 기반 서버 사이드 JSON 구조화 로깅. 개발 환경에서 pino-pretty로 가독성 확보. Notion API 에러 발생 시 pageId, 상태 코드, 에러 유형 기록 |
| **NFR-004** | 에러 트래킹   | (MVP 이후) Sentry 등 에러 트래킹 서비스 연동. 현재는 logger.error 일관 사용으로 대비                                                       |

이 서비스는 자체 데이터베이스를 사용하지 않습니다. 모든 데이터는 노션 API에서 실시간으로 조회합니다.

### 노션 데이터베이스 스키마 (F010)

상세 스키마 정의는 [@/docs/guides/notion-schema.md](./guides/notion-schema.md) 참조.

`invoices`, `items` 2개의 데이터베이스를 사용하며, `items.invoice` Relation으로 연결합니다.

## 기술 스택 (현재 프로젝트 구성 기준)

### 프론트엔드 프레임워크

- **Next.js 15.5.14** (App Router + Turbopack) - React 풀스택 프레임워크, SSR로 노션 API 서버사이드 조회
- **TypeScript 5** - 타입 안전성 보장, 노션 API 응답 타입 정의
- **React 19.1.0** - UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** (설정 파일 없는 새로운 엔진) - 견적서 레이아웃 스타일링
- **shadcn/ui** (new-york style) - Badge, Button, Table, Separator 컴포넌트 활용
- **Lucide React** - 다운로드 아이콘 등

### 폼 & 검증

- **React Hook Form** + **@hookform/resolvers** - 폼 상태 관리 (로그인 폼 등)
- **Zod** - 노션 API 응답 데이터, 환경변수, 인증 입력 유효성 검증

### 인증 & 보안

- **jose** - JWT 발급/검증 (HS256, Edge Runtime 호환)
- **bcryptjs** - 비밀번호 해싱 (SALT_ROUNDS=12)
- **@vercel/kv** - 관리자 비밀번호 해시 저장 (Redis 기반)

### 외부 API 연동

- **Notion API** (`@notionhq/client`) - 견적서 데이터 소스. 서버 사이드에서만 호출 (API 키 보호). `src/lib/notion.ts`에 클라이언트 초기화, 데이터 조회, 파싱, 오류 처리 통합 구현 완료
- **Resend** (`resend` v6.10.0) - 이메일 발송 서비스. 견적서 링크 이메일 전송, HTML/텍스트 이중 템플릿 지원

### PDF 생성

- **브라우저 인쇄 API (`window.print`)** - MVP 단계에서는 CSS `@media print` 스타일 최적화로 PDF 출력 지원 (별도 라이브러리 없이 구현)

### 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼, 환경 변수로 NOTION_API_KEY 관리

### 운영 & 모니터링

- **pino** - 구조화된 JSON 로깅 (서버 사이드, NFR-003)
- **Next.js middleware** - Rate Limiting 및 요청 로깅 (NFR-002)
- **React cache + unstable_cache** - Notion API 호출 최소화 (NFR-001)

### 패키지 관리

- **npm** - 의존성 관리

## 노션 API 연동 구현 메모

> **구현 완료**: `src/lib/notion.ts` 단일 파일에 클라이언트 초기화, 데이터 조회, 파싱, 오류 처리 통합 구현됨.

```
환경 변수 (필수)
- NOTION_API_KEY: 노션 통합(Integration) 시크릿 키
- NOTION_DATABASE_ID: invoices 데이터베이스 ID
- NOTION_ITEMS_DATABASE_ID: items 데이터베이스 ID

데이터 조회 흐름 (src/lib/notion.ts → getQuote 함수)
1. /quote/[pageId] 접속
2. Next.js 서버 컴포넌트에서 notion.pages.retrieve({ page_id: pageId }) 호출
3. notion.databases.query({
     database_id: NOTION_ITEMS_DATABASE_ID,
     filter: { property: "invoice", relation: { contains: pageId } },
     sorts: [{ property: "sort_order", direction: "ascending" }]
   }) 로 연결된 항목 조회 (Promise.all로 병렬 실행)
4. 응답 데이터를 Quote 타입으로 변환
5. 클라이언트에 렌더링

오류 처리 (구현 완료)
- APIResponseError (404): 존재하지 않는 페이지 → { error: "NOT_FOUND" } 반환 → notFound()
- APIResponseError (401): API 키 문제 → { error: "UNAUTHORIZED" } 반환 → Error throw
- 기타 예외: { error: "API_ERROR" } 반환 → Error throw
- 유효기간 초과: 페이지는 표시하되 만료 배너 표시 (app/quote/[pageId]/page.tsx에서 처리)
```
